# 개발 회고

## 잘한 점

- 백엔드, 프론트엔드, DB, 문서화까지 하나의 MVP 흐름으로 연결했습니다.
- JWT 인증과 비밀번호 해싱을 적용해 기본 보안을 고려했습니다.
- Docker Compose와 PowerShell 스크립트로 로컬 실행 과정을 정리했습니다.
- Flutter 앱을 탭 구조로 분리해 유지보수하기 쉽게 만들었습니다.
- 설정에서 다크/라이트 테마를 전환할 수 있도록 공통 색상 토큰을 정리했습니다.

## 어려웠던 점

- Windows/PowerShell 환경에서 한글 인코딩이 깨지는 문제가 있었습니다.
- Flutter Windows 빌드와 경로/캐시 문제가 발생해 빌드 환경을 정리해야 했습니다.
- MVP 범위를 지키면서도 포트폴리오에 보일 정도의 UI 완성도를 맞추는 균형이 필요했습니다.

## 개선한 점

- 깨진 한글 문서를 UTF-8 기준으로 재작성했습니다.
- 앱 UI를 홈/친구/피드/기록/설정으로 분리했습니다.
- 다크모드뿐 아니라 일반 라이트 버전도 추가했습니다.
- 테스트 문서를 최신 검증 결과 기준으로 업데이트했습니다.

- 피드 좋아요/댓글 구현
- 실제 GPS 경로 저장
- Android Emulator 및 실기기 QA
