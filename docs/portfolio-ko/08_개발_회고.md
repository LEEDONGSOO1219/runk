# 개발 회고

## 진행 결과

런크 MVP는 백엔드, DB, Flutter 기본 화면, Android Studio용 프로젝트 구조, GitHub 업로드까지 완료했다.

현재는 1단계 MVP의 기본 골격과 핵심 API가 동작하는 상태이며, 실제 Android Emulator에서 앱 플로우를 검증하는 단계가 남아 있다.

## 잘한 점

### 1. MVP 범위를 명확히 제한

초기부터 친구, 랭킹, GPS, AI 추천을 모두 구현하려고 하지 않고 회원가입, 기록 저장, 피드, 프로필에 집중했다.

### 2. 실행환경을 문서화

Docker, FastAPI, Flutter 실행 과정을 스크립트와 문서로 정리하여 이후 개발을 이어가기 쉽게 만들었다.

### 3. 실제 API 검증 수행

단순히 코드 작성에서 끝내지 않고 회원가입, 러닝 기록 저장, 피드 조회까지 smoke test로 검증했다.

### 4. 의존성 문제 해결

`passlib`와 `bcrypt` 버전 호환 문제를 발견하고 `bcrypt==4.0.1`로 고정하여 회원가입 오류를 해결했다.

## 어려웠던 점

### 1. Windows 개발환경 설정

Python, Docker, WSL, Flutter가 모두 필요해 초기 환경 구성이 복잡했다.

### 2. Docker 권한 문제

Docker Desktop 설치 과정에서 기존 폴더 권한 문제가 발생했고, 관리자 PowerShell에서 소유권과 WSL 업데이트를 처리해야 했다.

### 3. Android/Flutter 실행환경 분리

Chrome, Windows desktop, Android Emulator에서 백엔드 주소가 달라질 수 있어 `API_BASE_URL`을 주입하는 구조로 변경했다.

## 개선할 점

- Android Emulator에서 실제 앱 플로우를 검증하고 화면 캡처를 추가한다.
- UI를 포트폴리오용으로 더 정돈한다.
- API 오류 응답을 사용자에게 더 친절하게 표시한다.
- 테스트 케이스를 인증 실패, 입력값 오류, 빈 피드 상태까지 확장한다.
- 향후 Alembic을 도입해 DB 마이그레이션을 관리한다.
