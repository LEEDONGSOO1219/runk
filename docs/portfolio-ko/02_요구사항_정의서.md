# 요구사항 정의서

| --- | --- | --- |

| FR-002 | 사용자는 이메일과 비밀번호로 로그인할 수 있다. | 완료 |
| FR-003 | 앱 재시작 후 로그인 세션을 복원할 수 있다. | 완료 |
| FR-004 | 사용자는 러닝 거리, 시간, 날짜, 메모를 저장할 수 있다. | 완료 |

| FR-006 | 사용자는 내 러닝 기록을 조회할 수 있다. | 완료 |
| FR-007 | 사용자는 피드에서 최신 러닝 기록을 볼 수 있다. | 완료 |

## 비기능 요구사항

| ID | 요구사항 | 상태 |
| --- | --- | --- |
| NFR-001 | 비밀번호는 평문으로 저장하지 않는다. | 완료 |
| NFR-002 | API 입력값은 서버에서 검증한다. | 완료 |
| NFR-003 | JWT secret은 환경 변수로 관리한다. | 완료 |
| NFR-004 | 개발 DB는 Docker로 재현 가능해야 한다. | 완료 |
| NFR-005 | Flutter analyze/test를 통과해야 한다. | 완료 |
| NFR-006 | Windows 디버그 빌드가 가능해야 한다. | 완료 |

## MVP 제외 범위

- 실제 친구 관계 DB/API
- 실시간 채팅
- GPS 좌표 저장
- 좋아요/댓글 API
- 배포 자동화
