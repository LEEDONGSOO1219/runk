## 사용자 시나리오

### 시나리오 1. 신규 사용자가 러닝 기록을 저장한다

1. 사용자가 앱을 실행한다.
2. 이메일, 닉네임, 비밀번호로 회원가입한다.
3. 로그인 상태가 된다.
4. 러닝 기록 추가 화면으로 이동한다.
5. 거리, 시간, 날짜, 메모를 입력한다.
6. 기록을 저장한다.
7. 피드에서 새 기록을 확인한다.

### 시나리오 2. 기존 사용자가 피드를 확인한다

1. 사용자가 이메일과 비밀번호로 로그인한다.
2. 피드 화면으로 이동한다.
3. 최신 러닝 기록 리스트를 확인한다.
4. 프로필 화면에서 자신의 계정 정보를 확인한다.

## 제약사항

- MVP에서는 친구 관계 기반 피드 필터링을 구현하지 않는다.
- MVP에서는 실시간 위치 공유를 구현하지 않는다.
- MVP에서는 실제 배포 서버 대신 로컬 개발환경을 기준으로 구현한다.
