# DB 설계서

## ERD 개요

```text
users 1 ─── N running_records
```

사용자 한 명은 여러 개의 러닝 기록을 가질 수 있다.

## users 테이블

| 컬럼명 | 타입 | 제약조건 | 설명 |
| --- | --- | --- | --- |
| id | Integer | PK | 사용자 ID |
| email | String(255) | Unique, Index | 이메일 |
| username | String(50) | Unique, Index | 닉네임 |
| hashed_password | String(255) | Not Null | 해시된 비밀번호 |
| created_at | DateTime | Default now | 생성 시각 |

## running_records 테이블

| 컬럼명 | 타입 | 제약조건 | 설명 |
| --- | --- | --- | --- |
| id | Integer | PK | 러닝 기록 ID |
| user_id | Integer | FK, Index | 작성자 ID |
| distance_km | Float | Not Null | 거리(km) |
| duration_seconds | Integer | Not Null | 러닝 시간(초) |
| pace_seconds_per_km | Integer | Not Null | 1km당 페이스 |
| run_date | Date | Not Null | 러닝 날짜 |
| memo | String(255) | Nullable | 메모 |
| created_at | DateTime | Default now | 생성 시각 |

## 설계 의도

### 사용자와 기록 분리

사용자 정보와 러닝 기록을 분리하여 한 사용자가 여러 기록을 저장할 수 있도록 1:N 관계로 설계했다.

### 페이스 저장

페이스는 거리와 시간으로 계산할 수 있지만, 피드와 랭킹에서 자주 조회될 가능성이 있어 저장 컬럼으로 두었다.

### 추후 확장

2단계와 3단계에서는 아래 테이블 추가를 고려한다.

| 테이블 | 목적 |
| --- | --- |
| friendships | 친구 관계 |
| ranking_snapshots | 주간/월간 랭킹 |
| running_routes | 러닝 경로 |
| route_points | GPS 좌표 목록 |
