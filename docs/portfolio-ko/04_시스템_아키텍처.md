# 시스템 아키텍처

## 전체 구조

```text
Flutter App
    |
    | REST API
    v
FastAPI Backend
    |
    | SQLAlchemy ORM
    v
MySQL Database
```

## 구성 요소

| 구성 요소 | 역할 |
| --- | --- |
| Flutter App | 사용자 화면, 입력 처리, API 호출 |
| FastAPI Backend | 인증, 러닝 기록 저장, 피드 조회 API 제공 |
| MySQL | 사용자 정보와 러닝 기록 저장 |
| Docker Compose | 개발용 MySQL 컨테이너 실행 |
| Android Studio | Android 앱 실행 및 에뮬레이터 관리 |

## 백엔드 구조

```text
backend/app
├── core/
│   ├── config.py
│   └── security.py
├── db.py
├── dependencies.py
├── main.py
├── models.py
└── schemas.py
```

| 파일 | 설명 |
| --- | --- |
| `main.py` | FastAPI 앱과 라우터 정의 |
| `models.py` | SQLAlchemy DB 모델 |
| `schemas.py` | Pydantic 요청/응답 모델 |
| `db.py` | DB 엔진과 세션 관리 |
| `dependencies.py` | 인증 사용자 조회 |
| `security.py` | 비밀번호 해싱, JWT 발급 |
| `config.py` | 환경 변수 설정 |

## 프론트엔드 구조

```text
frontend/lib
├── models/
├── screens/
├── services/
├── widgets/
└── main.dart
```

| 폴더 | 설명 |
| --- | --- |
| `models` | API 응답 데이터 모델 |
| `screens` | 화면 단위 UI |
| `services` | API 클라이언트 |
| `widgets` | 재사용 UI 컴포넌트 |

## 인증 흐름

```text
회원가입/로그인 요청
    -> FastAPI에서 사용자 검증
    -> JWT access token 발급
    -> Flutter 앱에서 token 보관
    -> 인증 API 호출 시 Authorization header 사용
```

## Android Emulator 네트워크

Android Emulator에서 PC의 `localhost`는 에뮬레이터 자신을 의미한다. 따라서 PC에서 실행 중인 FastAPI 서버에 접근하려면 아래 주소를 사용한다.

```text
http://10.0.2.2:8000
```

Flutter 실행 시 아래 값을 전달한다.

```powershell
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000
```
